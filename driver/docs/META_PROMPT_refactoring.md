## [프로젝트 맥락]

### 무엇인가
`pixel-agents` 의 신규 `driver/` 폴더. **Claude Code 없이** OpenRouter 작은 모델(SLM)로
N명 에이전트를 구동해, 기존 픽셀 오피스 서버가 이해하는 훅 신호로 변환 → 캐릭터로 시각화한다.
이미 1차 PoC가 완성되어 동작 중이다(단위 테스트 58개 통과, 실서버 라이브 검증 완료).
이번 작업은 그 위에 **추가기능을 얹는 것**이다(기존 동작은 보존).

### 현재 구조 (`driver/src/`)
| 파일 | 역할 |
|------|------|
| `index.ts` | 진입점 + 오케스트레이션(`runAll`/`stopAll`), SIGINT 종료 |
| `config.ts` | 에이전트 정의(이름·모델) + 환경변수 설정 로드/검증 |
| `agent.ts` | 행동 루프(`start`/`tickOnce`/`runLoop`/`stop`) |
| `openrouter.ts` | OpenRouter 호출 + `{action,target,reason}` 견고한 파싱 |
| `actions.ts` | action → 훅 tool_name + 한국어 문구 매핑 |
| `office.ts` | 서버 연동(server.json 읽기, 훅 POST, JSONL 쓰기, 해시) |
| `logger.ts` | 한국어 컬러 로그 |
| `backoff.ts` | 지수 백오프 |
| `semaphore.ts` | 동시 호출 제한 |
| `scripts/smoke.ts`, `scripts/integration-check.ts` | 체험/자동점검 스크립트 |

### 동작 계약 (추가기능이 깨면 안 되는 것)
- 서버 훅 입구 `POST /api/hooks/claude`(Bearer 인증), `~/.pixel-agents/server.json`(`token` 필드).
- 행동 4종 enum: `read|write|run|rest` → `Read/Edit/Bash/(Stop)`.
- 캐릭터 등장: 워크스페이스 정합 또는 Watch All Sessions + 브라우저 선연결(새로고침 금지).
- 비용/레이트리밋 정책(ADR-011): 루프 주기 하한·동시성 세마포어·지수 백오프.

### [기능 후보] (참고용 — AI가 제안 시 활용)
- **토큰/비용 표시**: OpenRouter 응답의 usage(입력/출력 토큰)를 집계해 로그·요약으로 출력.
- **페르소나**: 에이전트별 시스템 프롬프트/말투로 행동·reason 다양화(config 확장).
- **행동 다양화**: action 종류 확장(회의/대화/협업 등) — 단, 서버가 아는 tool_name 매핑은 유지.
- **설정 외부화**: `agents.json` 등 외부 파일로 에이전트·정책 분리(+스키마 검증·친절한 한국어 에러).
- **실작업화(단계적)**: read부터 실제 파일 읽기(안전 범위), write/run은 드라이런으로 시작.
- **상태 대시보드**: 터미널에 에이전트별 현재 상태/호출수/백오프/토큰을 주기적으로 요약 출력.
- **모델 폴백**: 404/402/429 시 대체 모델로 자동 전환(가용 모델 목록 기반).
- **한국어 조사 엔진**: 받침 판별로 을/를·이/가·은/는 자동 선택(문구 자연스러움 향상).
- **Provider 승격(2단계)**: `server/src/providers/hook/openrouter/` 정식 provider(서버 수정 허용 범위·근거를 ADR로 명시).

## [작업 원칙 (반드시 반영)]

1. **TDD 방법론**: 테스트 먼저(Red) → 구현(Green) → 리팩터(Refactor). 각 단계 완료 기준에 테스트 통과 포함
2. **ADR 문서화**: 모든 설계 결정에 ADR 작성. `driver/docs/adr/ADR-NNN-제목.md`
3. **한국어 원칙**: 주석·로그·문서 전부 한국어. 변수명/함수명만 영어
4. **상세 주석**: 모든 함수에 목적·입출력·의존성을 한국어로 설명
5. **기존 동작 보존**: 추가기능은 **기존 58개 테스트 그린 유지**가 안전망(회귀 0). 새 기능엔 새 테스트.
6. **계층/제약 준수**: 1단계는 `server/`·`webview-ui/`·`core/`·`adapters/vscode/` **무수정**.
   서버 수정이 필요한 기능(예: Provider 승격)은 **별도 단계로 분리**하고 ADR로 근거를 남긴다.
7. **비용 고려**: 추가 호출이 생기면 루프 주기·동시성·백오프(ADR-011) 정책 안에서 설계.
8. **기록 일관성**: 각 단계 종료 시 `TEST_LOG.md` 갱신 + `docs/reports/STAGE-N-완료보고서.md` 작성
   (기존 프로젝트의 단계별 보고서·테스트 로그 관례를 그대로 잇는다).

## [제약 및 비목표]
- ADR 번호는 기존 **ADR-012 다음(013~)** 으로 이어서 매긴다.
- 한 번에 큰 변경 금지: 기능을 작은 단계로 쪼개 **매 단계 테스트 그린**을 유지한다.
- 기능 단계는 서로 독립적으로 진행·검증 가능하도록 설계한다(한 기능 실패가 다른 기능을 막지 않음).

---

## [최종 완료 기준]
1. 추가기능이 동작하고, 각 기능에 **새 테스트**가 붙는다(TDD).
2. **기존 58개 테스트 + 신규 테스트 전부 그린**, 타입체크 통과.
3. 모든 설계 결정에 **ADR(013~)** 가 있고, 단계별 **완료 보고서 + TEST_LOG** 갱신.
4. 비용/레이트리밋 정책 위반 없음, 기존 동작 회귀 없음.
