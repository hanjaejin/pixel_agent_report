# ADR-009: 프로세스 종료 시 세션 정리(SessionEnd 전송)

- 상태: 채택(Accepted)
- 날짜: 2026-06-26

## 맥락

드라이버가 Ctrl+C 로 종료될 때 아무 신호도 안 보내면, 오피스에는 캐릭터가 "유령"으로
남는다(서버의 stale 정리(30초)를 기다려야 사라짐).

## 결정

- `SIGINT`/`SIGTERM` 수신 시 전원에 대해 `stop()`(= `SessionEnd(exit)` 훅)을 보낸다.
- 중복 종료를 막기 위해 `stopping` 가드를 둔다.
- `stopAll` 은 `Promise.allSettled` 로 한 에이전트의 종료 실패가 다른 종료를 막지 않게 한다.

## 결과

- 종료 즉시 캐릭터가 깔끔하게 퇴장한다.
- 종료 경로가 견고하다(부분 실패 허용).
