# ADR-010: 에이전트 격리 — 한 명의 실패가 전파되지 않음

- 상태: 채택(Accepted)
- 날짜: 2026-06-26

## 맥락

N명을 한 프로세스에서 병렬 구동한다. 한 에이전트의 루프가 예외로 죽어도 나머지는
계속 일해야 한다(데모 견고성).

## 결정

- `runAll` 은 `Promise.allSettled(agents.map(a => a.runLoop()))` 로 각 루프를 격리한다.
- 거부된 루프는 한국어 로그로 보고하되, 전체를 중단시키지 않는다.
- 각 에이전트는 자기 `AbortController`/세션을 가져 상태를 공유하지 않는다.
- LLM 호출 실패는 에이전트 내부에서 rest 폴백 + 백오프로 흡수(ADR-004/007), 루프 종료까지
  가지 않는다 → 격리는 이중 안전망의 바깥 그물.

## 결과

- 모델 하나가 자주 실패해도 다른 캐릭터는 계속 살아 움직인다.
- 최종 완료 기준(N명 독립 동작)을 구조적으로 보장.
