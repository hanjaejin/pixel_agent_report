# ADR-011: 비용·레이트리밋 정책 (주기·동시성·백오프)

- 상태: 채택(Accepted)
- 날짜: 2026-06-26

## 맥락

N명 에이전트가 짧은 루프로 OpenRouter 를 호출하면 비용이 빠르게 늘고 429(레이트리밋)를
맞기 쉽다. 작은 모델이라도 호출량 통제가 필요하다.

## 결정

세 가지 방어선을 둔다(모두 환경변수로 조절, 기본값 보수적).

1. **루프 주기 하한**: `PIXEL_LOOP_INTERVAL_MS`(기본 4000ms)를 **최소 1000ms 로 클램프**.
   너무 잦은 호출을 구조적으로 차단.
2. **동시 호출 상한(세마포어)**: `PIXEL_MAX_CONCURRENCY`(기본 2). 전 에이전트가 **하나의
   세마포어를 공유**해 동시에 OpenRouter 를 때리는 수를 제한 → 순간 부하/비용 억제.
3. **지수 백오프**: 호출 실패(429/5xx 등) 시 `nextBackoffMs(attempt, base, max)` =
   `min(max, base*2^attempt)`. `PIXEL_BACKOFF_BASE_MS`(2000) ~ `PIXEL_BACKOFF_MAX_MS`(30000).
   연속 실패할수록 간격이 늘고, 성공하면 즉시 초기화. **백오프 동안 캐릭터는 `Stop`(대기)로
   표시**되어 사용자가 "쉬는 중"임을 본다.

## 결과

- 비용/레이트리밋이 통제되고, 일시적 429 에도 크래시 없이 우아하게 대기한다.
- 정책 수치는 운영 중 환경변수로 즉시 조절 가능(코드 수정 불필요).
- 관련: 에이전트 격리(ADR-010)는 백오프가 한 에이전트에 묶여도 다른 에이전트를 막지 않게 한다.
