# ADR-008: PoC 는 실제 파일을 건드리지 않는다 (행동은 데모용)

- 상태: 채택(Accepted)
- 날짜: 2026-06-26

## 맥락

작은 모델로 실제 코드 작업(파일 읽기/수정/명령 실행)을 시키면 위험(파괴적 변경)하고
불안정하다. 1단계 PoC의 목표는 "캐릭터가 살아 움직이는 것"을 보여주는 것이다.

## 결정

- PoC 에서 read/write/run 은 **신호만 보낸다**(office 훅 POST). 실제 파일/명령은 실행하지 않는다.
- LLM 은 "무슨 일을 하는 척"할지(action+target+reason)만 생성하고, 드라이버는 그것을
  훅 신호 + 한국어 로그로 변환한다.
- 추후 실작업으로 교체할 수 있도록 행동 처리를 `actions.ts`(매핑)와 `agent.ts`(실행 흐름)로
  분리해 둔다. 실작업은 단계 3(선택)에서 `tickOnce` 의 행동 분기만 바꾸면 된다.

## 결과

- 안전하고 결정적이며 데모에 충분하다.
- 실작업화는 office 신호 흐름을 바꾸지 않고 행동 핸들러만 교체해 달성 가능.
