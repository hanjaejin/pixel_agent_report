# ADR-005: 에이전트별 모델·키 주입 전략

- 상태: 채택(Accepted)
- 날짜: 2026-06-26

## 맥락

최종 완료 기준 중 하나는 "에이전트마다 다른 OpenRouter SLM 모델 사용"이다.
모델은 에이전트별로 다르고, API 키는 보통 전역 1개지만 향후 에이전트별로 다를 수 있다.

## 결정

- `createOpenRouter({ apiKey, ... })` 는 **키 단위**로 클라이언트를 만든다.
- `decide(model, ...)` 는 **호출 시점에 모델을 인자로** 받는다 → 한 키로 여러 모델 사용 가능.
- 에이전트 정의(`config.ts`, 단계 6)는 `{ name, model }` 을 갖고, 전역 키
  `OPENROUTER_API_KEY` 를 공유한다. 에이전트별 키가 필요하면 config 항목에 `apiKey?` 를 추가해
  해당 에이전트만 별도 클라이언트를 쓰도록 확장한다(현재는 미사용, 확장점만 확보).

## 결과

- 키 1개 + 모델 N개 조합을 기본으로 지원(가장 흔한 사용 형태).
- 키 분리 요구가 생겨도 config 스키마만 늘리면 되고 클라이언트 코드는 그대로.
