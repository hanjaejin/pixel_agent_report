# ADR-006: 한국어 문구는 드라이버 고정 템플릿, 모델은 reason 한 줄만

- 상태: 채택(Accepted)
- 날짜: 2026-06-26

## 맥락

화면(픽셀 오피스)의 캐릭터 동작과 터미널 로그가 1:1로 대응해야 한다(최종 완료 기준 3).
한국어 업무 문구 전체를 작은 모델이 생성하게 하면, 어휘/형식이 들쭉날쭉해지고
이모지·문장 구조가 깨질 수 있다.

## 결정

- action 별 **한국어 문구와 이모지는 `actions.ts` 의 고정 템플릿**이 만든다.
  - read → `📖 ... 살펴보고 있어요`, write → `✏️ ... 수정하는 중이에요`,
    run → `⚙️ ... 명령을 실행하고 있어요`, rest → `☕ 잠깐 쉬는 중이에요`
- 모델은 `reason`(한 줄)만 생성한다(ADR-004). reason 은 로그의 부가 설명으로만 쓰고,
  화면 동작을 가르는 tool_name 매핑에는 영향을 주지 않는다.
- `target` 이 비면 일반 문구로 폴백해 `undefined` 노출을 방지한다.

## 결과

- 로그 ↔ 화면 동작의 1:1 대응이 결정적으로 보장된다.
- 한국어 조사(을/를) 등은 PoC 수준의 단순 템플릿으로 처리(완벽한 형태소 처리는 비목표).
