# ADR-016: 에러 분류와 모델 폴백(영구 vs 일시)

- 상태: 채택(Accepted)
- 날짜: 2026-06-27
- 관련: PLAN_2 F4

## 맥락

라이브 테스트에서 모델 ID 오류(404)·무료 한도(402)·인증(401)·레이트리밋(429)을 겪었다.
이전에는 모든 실패를 **지수 백오프**로만 처리해, 404/402처럼 **재시도해도 안 되는 에러**에도
계속 같은 모델로 헛되이 대기했다.

## 결정

OpenRouter 응답 실패를 두 갈래로 분류한다(`OpenRouterError.status`).

- **영구 에러(모델/인증)**: `400, 401, 402, 403, 404` → **대체 모델로 전환**.
  - `AgentConfig.fallbackModels`(= `agents.json`의 `fallbackModels`)를 주 모델 뒤에 붙여
    `[model, ...fallbackModels]` 목록을 만들고, 영구 에러 시 인덱스를 1 전진.
  - 전환 후 이번 턴은 `Stop`(대기)으로 마치고, 다음 tick부터 새 모델 사용.
  - 폴백을 모두 소진하면 짧게 대기(backoffBase)하고 rest.
- **일시 에러(레이트리밋/서버/네트워크/타임아웃/파싱)**: `429, 5xx`, 비-OpenRouterError →
  **같은 모델로 지수 백오프**(기존 ADR-011 정책 유지).

## 결과

- 잘못된/사용 불가 모델이어도 자동으로 대체 모델로 넘어가 **멈추지 않는다**.
- 일시적 429에는 그대로 백오프해 비용/레이트리밋을 보호한다.
- `OpenRouterError`로 상태코드를 구조화해 분류 로직이 명확하다(문자열 파싱 불필요).
- F5(런타임 모델 선택)에서 이 모델 목록/인덱스를 외부에서 바꾸도록 확장할 토대가 된다.
