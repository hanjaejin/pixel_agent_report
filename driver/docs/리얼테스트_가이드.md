# 🧪 리얼테스트 가이드 — 신규기능 F1~F6 하나하나 확인

> 초보자도 따라 할 수 있게, **기능별로 "무엇을 하면 무엇이 보이는지"**를 정리했습니다.
> 순서: 빌드 → 설정 → 서버 → 브라우저 → 드라이버 → 기능별 확인.

---

## 0. 준비 (최초 1회)

### 0-1. 빌드 (F5 서버 라우트 + F6 웹뷰 반영)
저장소 루트에서:
```
npm run build
```
> 이미 빌드돼 있으면 생략 가능. (서버/웹뷰 코드를 바꿨다면 다시 실행)

### 0-2. 에이전트 설정 파일 만들기 (F1)
`driver` 폴더에서 예시를 복사:
```
cd driver
copy agents.example.json agents.json
```
→ `agents.json` 에 김대리/박사원/이주임이 **persona·skillFile·fallbackModels** 와 함께 들어 있습니다.

### 0-3. 열쇠 + (선택)언어
`driver\.env` 에 `OPENROUTER_API_KEY=sk-or-...` 입력(이미 했으면 OK).
드라이버 로그를 영어로 보고 싶으면(F6) `.env` 에 `PIXEL_LANG=en` 추가(기본은 한국어).

---

## 1. 서버 + 브라우저 켜기

**창 A (서버):**
```
cd <저장소 루트>
node dist/cli.js --port 3100
```
**브라우저:** `http://localhost:3100` → 설정(⚙️)에서 **Watch All Sessions ON** → (새로고침 금지)

**창 B (드라이버):**
```
cd <저장소 루트>\driver
npm run start:env
```
→ 김대리·박사원·이주임 등장. 이제 기능별로 확인합니다.

---

## ✅ F1 — agents.json 외부화

**확인 1 (로드):** 창 B 시작 로그에 `👥 에이전트 3명...` + 각 캐릭터 출근. agents.json 의 이름이 그대로 뜸.
**확인 2 (수정 반영):** 창 B `Ctrl+C` → `agents.json` 에서 `"김대리"` 를 `"최부장"` 으로 바꿔 저장 → `npm run start:env` 다시 → 로그에 **최부장** 등장.
**확인 3 (친절한 한국어 에러):** `agents.json` 에서 한 에이전트의 `"model"` 줄을 지우고 저장 → 다시 실행 → 빨간 한국어 에러
`agents[N](이름).model 이 비어 있거나 문자열이 아닙니다.` 로 **즉시 중단**. (확인 후 원복)

---

## ✅ F2 — 페르소나(성격/말투)

**확인:** 창 B 로그의 `(이유)` 부분이 성격을 반영하는지 본다.
- 김대리(꼼꼼) → "먼저 확인하려고…" 류 신중한 이유
- 박사원(열정 신입) → 적극적인 이유
- 이주임(느긋) → 여유로운 이유

> 실제 AI라 매번 다르지만, 성격 경향이 드러나면 OK. persona 를 바꾸면(예: "급한 성격") 이유 톤이 달라짐.

---

## ✅ F3 — SKILL.md 참조

**확인:** 김대리는 `skillFile: skills/example-김대리.md`(="작업 전 항상 먼저 읽어라") 가 붙어 있어,
**read(📖)로 시작**하는 경향이 강해진다. 창 B 에서 김대리의 첫 행동/이유가 "읽기/확인"인지 관찰.
**대조:** `agents.json` 에서 김대리의 `skillFile` 을 지우면 그 경향이 약해짐(원복 권장).

---

## ✅ F4 — 모델 폴백(자동 대체)  ⭐가장 확실한 테스트

**설정:** `agents.json` 에서 한 에이전트를 일부러 **없는 모델 + 유효 폴백**으로:
```json
{ "name": "박사원", "model": "fake/does-not-exist", "fallbackModels": ["qwen/qwen-2.5-7b-instruct"] }
```
저장 → 창 B 다시 실행.
**확인:** 박사원 로그에
`⚠️ 모델 fake/does-not-exist 사용 불가(HTTP 404) → qwen/qwen-2.5-7b-instruct 로 전환합니다`
가 뜨고, 그 뒤로 **멈추지 않고 계속 일함**. (확인 후 원복)

---

## ✅ F5 — 화면에서 모델 선택  ⭐화면 조작

**확인:**
1. 드라이버가 도는 동안 브라우저 **설정(⚙️) 열기**.
2. 아래쪽 **"Agent Models"** 섹션에 에이전트별 드롭다운(현재 모델)이 보인다.
3. 한 에이전트의 모델을 **다른 값으로 변경**.
4. 창 B 로그를 보면 그 에이전트가 **다음 행동부터 새 모델로 호출**(F4 와 함께 보면, 가짜 모델로 바꾸면 폴백/대기 발생).

> 드롭다운이 안 보이면: 드라이버가 떠 있고(상태 보고 1.5s 주기), 브라우저를 새로고침하지 않았는지 확인.
> 새로고침했다면 드라이버를 재시작하면 다시 표시됨.

---

## ✅ F6 — 한↔영 전환

**화면(웹뷰) 언어:**
1. 브라우저 **설정(⚙️)** 열기 → 맨 위 **"한국어 (Korean)"** 체크.
2. 설정 안의 라벨들이 **한국어로 전환**(설정/소리 알림/모든 세션 감시/에이전트 모델…). 새로고침해도 유지(localStorage).

**드라이버 로그 언어:**
1. 창 B `Ctrl+C` → `driver\.env` 에 `PIXEL_LANG=en` 추가 → `npm run start:env`.
2. 업무 로그가 영어로: `🚪 Clocked in`, `📖 Looking at config.ts`, `☕ Taking a short break` 등.
3. 한국어로 되돌리려면 `PIXEL_LANG` 줄 삭제(또는 `=ko`).

> 참고: 오피스 캔버스의 활동 라벨("Reading…")은 서버 공유 부분이라 영어로 유지됩니다(설계상 범위 밖).
